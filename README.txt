Микросервис для отправки сообщений на мессенджеры по API
Версия: v1.0

url для доступа к API: http://[hostname]/messengers/api/v1.0/
url состоит из названия приложения, версии API (версия API добавлена, чтобы в случае обновления можно было сохранить
старую версию для совместимости) и функции приложения.

Функции приложения:

send_message:
В приложении реализована функция отправки сообщений: send_message
по адресу: http://[hostname]/messengers/api/v1.0/send_message
Функция принимает POST запрос со входным json
json имеет следующие параметры:
'id': [список id на которые необходимо отправить сообщения],
'messengers': [список мессенджеров на которые необходимо отправить сообщения],
'text': текст сообщения,
не обязательно:
'date': дата и время отправки сообщения в формате (ДД.ММ.ГГГГ ЧЧ:ММ)
В случае если json указана дата сообһение сохраняется в базу данных для отложенной отправки
В случае если сообщения не удалось отправить, сообщения сохраняются для повторной отправки

check_database:
Функция проверяет наличее сообщений в базе данных на наличие сообщений время отправки которых меньше или равно текущему.
Функция запускается раз в минуту.
В случае если в базе данных есть сообщения функция производит их отправку на соответсвующие мессенджеры.

Дополнительные модули:

Модули Telegram, Viber и WhatsApp:
Данные модули служат для имитации отправки сообщений на мессенджеры.
В качестве имитации функции выводят на консоль id, текст сообщения и названия мессенджера.

Модуль DatabaseUpdater:
Модуль предназначен для управления базой данных.
Используется библиотека peewee и база данных MySQL. Проект не требователен к базе данных, поэтому выбраны эти ресурсы,
так как они довольно простые, не требует сложной настройки и позволябт легоко управлять базой данных.
Модуль создает базу данных MessageDB.db с таблицей Database котороя содержит следующие поля:
id - id
text - текст сообщения
messenger - мессенджер
date - время для отправки сообщения
